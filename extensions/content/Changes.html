Version 4.3.0

üéâ New Features

  • Added ClampMagnitude .
  • Added FindByType .
  • Added methods to convert between colors and hexadecimal strings: TryParseHex, ParseHex , ToRGBHex ,ToRGBAHex .
  • Added a common base class for properties that validate their values ValidationAttribute , and added different ways to do the validation through flags in PropertyDrawerData .
  • Added ValidateNotNullAttribute, ValidatePositiveAttribute, ValidateNonNegativeAttribute, ValidateMagnitudeRangeAttribute, and deprecated WarningIfNullAttribute, PositiveAttribute, NonNegativeAttribute.
  • HighlightAttribute now supports a color parameter, and is drawn better in the non-pro skin of Unity.
  • WarningIfNullAttribute now supports a color parameter.
  • Added PropertyDrawerData that can be used to set global properties used by property drawers, such as default colors and list retrievers.
  • Added property drawer support for drawing popups of various types, including strings, integers, colors, tags, and layers. See subclasses of PopupListAttribute and PopupListPropertyDrawer<T> .
  • Changed all relevant mono behaviours and scriptable objects to use the new validation attributes.
  • Added SeparatorAttribute and SeparatorDrawer to draw separators in the inspector.
  • Added SuppressReason that has justifications for suppressing certain warnings, and deprecated Warning.
  • Added Branding.Hex to provide string presentations of colors.
  • Added White and Black for consistency.
  • Added EditorOnly to mark symbols only available in the editor.

üêû Bug Fixes

  • Made ThrowIfNegative a extension method, like the other extension methods in this class.

üîß Engineering

  • Made InspectorList and all its subclasses obsolete for Unity 2020.1 and later, since Unity's new list provide similar (and more robust) functionality.

üß™ Samples

  • Changed all relevant samples to use the new validation attributes.

üìÑ Documentation

  • Expanded the documentation for Property Drawers.
