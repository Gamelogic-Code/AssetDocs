Classes

GameMove<TPoint, TPieceSettings>

A class that represents a change in state to the gameManager. Moves can change their state forwards and backwards, so that they can undo changes that they have made and restore the former state

GameRules<TPoint, TPieceSettings>

Class to handle general gameManager flow, moving between states, move logic and validation, and victory states. Attach victory rules to this object to have them processed when the gameManager progresses.

HumanPlayer<TPoint, TPieceSettings>

Class to represent a human player, who makes decisions on how to act. You will probably not use this class directly, and instead use the non-generic variant GridHumanPlayer.

PieceRule<TPoint, TPieceSettings>

Rule object describing how a given type of piece behaves. Implement these and attach them to PieceSettings<TPoint, TSelf> objects to describe behaviours for specific pieces.

You can disable the behaviour with enabled to temporarily disable a rule.

PieceSettings<TPoint, TSelf>

Class to contain settings for each piece. Attach rules for each piece as additional components to this script.

Player<TPoint, TPieceSettings>

Player base class.

TurnManager<TPoint, TPieceSettings>

VictoryRules<TPoint, TPieceSettings>

A component that represents a gameManager victory rule.

You can disable the behaviour with enabled to temporarily disable a victory rule.
