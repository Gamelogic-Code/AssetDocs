Version 2.1.0

🎉 New Features

  • Added Edit and Recompute buttons to the graph fields in the inspector.
  • Added colors to InspectableColorNode fields in the inspector.
  • Added various methods to parse colors from strings:
    • Parse
    • TryParse
    • ParseHex
    • TryParseHex
  • Added support for LAB colors (see for example ColorLAB ). This includes new mix modes, and nodes for sampling from LAB color gradients.
  • Added Gradient<TColor> and GradientKey<TColor> classes to represent gradients of colors in arbitrary color spaces.
  • Added an algorithm GenerateInAffineCircle to sample from an ellipse in the RGB color space.
  • Graphs can now be created under the Assets | Create | Gamelogic | Colors | Graph menu, or from the same place on the context menu in the project view.
  • Nodes can now be renamed, and have better default names.
  • Added new nodes:
    • AffineCircleNode
    • ConcatenateNode
    • TakeNode
    • TakeRandomNode
    • TakeAtIndicesNode
    • Mix2Node, which is to replace MixNode2. The latter was misnamed and has been made obsolete.
  • Made graph technology more re-usable. (This is mostly for internal convenience.)
    • Made the GraphWindow not depend on any colors-specific functionality, and introduced ColorGraphWindow to handle the colors-specific functionality.
    • Made GraphPropertyDrawer<TWindow> generic, and added a new ColorGraphPropertyDrawer to handle colors-specific functionality.
    • Made GraphEditor<TWindow> generic, and added a new ColorGraphEditor to handle colors-specific functionality.

🐞 Bug Fixes

  • Made repainting of scene and editor UI more robust when graph colors get recomputed.
  • Made editor assemblies only build for the editor.

🔧 Engineering

  • Updated to use Extensions 3.1.0.

📚 Examples

  • Improved the examples to make it clearer how colors are recomputed and assigned for new different use-cases, and to allow you to configure when these happen in the inspector.
