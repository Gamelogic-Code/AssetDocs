Introduction

Grids is a big library. Many of the classes are the machinery that makes it work, and you will only need them for advanced uses. This guide will point you to the most useful classes for general usage.

Section

GridPoint2 (T:Gamelogic.Grids2.GridPoint2)

Represents 2D grid locations, and defines the methods for point arithmetic. Points are how cells in a grid are identified, and form the basic element of grid algorithms.

IGrid2 (T:Gamelogic.Grids2.IGrid2)

This interface is generally how you work with grids. There are concrete implementations, but in general, you should work with this interface instead, and most algorithms are defined in terms of this interface. For example, a 2D grid of integers is IGrid{GridPoint2, int}.

In advanced usages of grids, you may also want to provide your own implementation of this interface.

Algorithms (T:Gamelogic:Grids2:Algorithms)

This class contains various algorithms that operate on grids, such as path finding, connected shapes, and range finding. Usually, these algorithms require knowledge of the cells (that is, the data in the grid). Algorithms that only work on the grid shape is contained in either T:Gamelogic.Grids2.IImplicitShape1orT:Gamelogic.Grids2.IExplicitShape1.

Map (T:Gamelogic.Grid2.Map)

This class defines methods for creating new maps, and transforming maps into new maps. Generally, you would only use this class if you create grids in code. For grids created with grid builders, maps are built from the node-based space maps and the round map setting in the grid builder. (See link)
